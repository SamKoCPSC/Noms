# NOMS-002: Initial Project Setup — Repository, Workspace & Infrastructure

**Status:** 🟢 Done
**Phase:** Pre-work
**Created:** 2026-05-09

## Description

Establish the foundational codebase and external infrastructure so that subsequent Phase 1 issues have something to build against. This is purely scaffolding — no product features, just the plumbing that makes development possible.

## Scope

### Project Scaffolding (Dioxus Fullstack)
- [x] Scaffold project using `dx init` with the fullstack template:
  ```bash
  cargo install dioxus-cli
  dx init --template fullstack --subtemplate Barebones --vcs none
  ```
  This produces a single-crate layout — one codebase that compiles to both native (SSR + server functions) and WASM (client):
  ```
  noms/
  ├── Cargo.toml          # Single crate, shared deps with platform features
  ├── Dioxus.toml         # Platform config (fullstack default)
  ├── src/main.rs         # Entry point (shared entry for client + server)
  └── assets/             # Static assets
  ```
- [x] Configure `Cargo.toml` with platform-specific feature flags (generated by `dx init`):
  ```toml
  [features]
  default = []
  server = ["dioxus/axum"]
  web    = ["dioxus/web"]
  ```
  Server-only dependencies (SQLx, aws-sdk-s3) are gated behind the `server` feature so they never compile into the WASM client binary.
- [x] Add foundational dependencies:
  - Dioxus fullstack (`dioxus` with `"fullstack"` + `"router"` features)
  - SQLx with PostgreSQL feature (server-only, gated by `[features]`)
  - `tracing` + `tracing-subscriber` (JSON structured logs)
  - `aws-sdk-s3` for Cloudflare R2 integration (server-only)
- [x] Organize source tree into modules:
  ```
  src/
  ├── main.rs           # App entry, routing config
  ├── pages/            # Route components (Home, RecipeList, RecipeDetail, etc.)
  ├── components/       # Reusable UI components
  ├── db/               # Database layer (SQLx types, queries — server-only via cfg)
  └── utils/            # Shared utilities
  ```
- [x] Configure `rustfmt` and `clippy` at crate level (`Cargo.toml` `[lints]`)

### Railway Infrastructure
- [x] Create Railway project with two services:
  - **App service** — Single Dioxus Fullstack binary (SSR, server functions, and static serving)
  - **PostgreSQL database** — Railway managed Postgres instance
- [x] Create **staging** and **production** environments (trunk-based):
  - **Staging** — auto-deploys on every push to `main`
  - **Production** — manual deploy via Railway "promote" from a staging build (preserves production config, swaps the binary)
  - Each environment has its own isolated PostgreSQL instance
- [x] Configure environment variables per environment:
  - **Staging**: `DATABASE_URL` (staging Postgres), R2 credentials, OAuth credentials with staging redirect URIs, `SESSION_SECRET` (unique)
  - **Production**: `DATABASE_URL` (production Postgres), R2 credentials, OAuth credentials with production redirect URIs, `SESSION_SECRET` (unique)
